## <a id="_6scps0zeif0j"></a>Technisch Onderzoek Researchproject Semi\-Supervised Plant Disease Classification met Burn in Rust op Edge Device

## <a id="_cqzr4alw1gca"></a>Welke data gebruik je?

Voor dit project wordt de PlantVillage dataset gebruikt, bestaande uit 61\.486 afbeeldingen van plantbladeren met 39 verschillende plantenziektes en gezonde variant\. De afbeeldingen zijn 256x256 pixels en bestaan uit gefotografeerde bladeren in geconditioneerde laboratoriumomgevingen\. Om een semi\-supervised setting te simuleren, wordt willekeurig 20\-30% van de dataset gelabeld aangemerkt, terwijl de rest als "ongelabeld" wordt behandeld\. Dit is realistischer voor praktische scenario's waarin handmatig labelen duur is\.

## <a id="_w32up6a6cmb3"></a>

## <a id="_hlugx2jsvoob"></a>Welke case ga je uitwerken?

De case richt zich op real\-time plantziekeclassificatie via lokale camera’s op de Jetson \(we gaan dit niet doen wnat ik heb geen zieke planten, we gaan hier gewoon afbeeldingen voor gebruiken uit de dataset\) In praktijk is er een camera dat een foto neemt van de bladeren, het model classifeert direct of er ziekte aanwezig is en welke\. Geen cloud of latency, puur lokaal op de edge device\. En per nieuwe foto dat we dan effectief correct labelen via het model gaan we het model verder op trainen\. Aka de semi\-supervised learning\. Het Model leert dus continu bij via pseudo\-labeling van high\-confidence voorspellingen\. en periodeek \(bv om de 500 nieuwe afbeeldingn\- wordt het model opnieuw getrained met de originele labeled data \+ het best zelfgenereerde pseudo\-labels\.   
  
Praktisch gaan we dit doen via cvat, een open labelingstool\.

## <a id="_lmvwfp51bkl2"></a>Welke vergelijkingscriteria onderzoek je?

- Modelprestatie: Accuracy en F1\-score op testdata, inclusief automatische labels\.
- Inference Snelheid: Milliseconds per frame op Jetson Orin Nano hardware – doel: <500ms per image, ideaal <200ms\. Dit is de belangrijkste metric voor edge devices, niet puur accuracy\.
- Resource Utilization: GPU VRAM usage, CPU usage, power consumption op Jetson\. Vergelijk welke retrainingfrequentie het beste balans geeft tussen model drift en computational cost\.
- Codekwaliteit: Overzichtelijkheid, efficiëntie en onderhoudsvriendelijkheid van de Rust\-implementatie\.
- Comparison: Burn vs\. candle\_core vs\. tch\-rs – welke library geeft beste trade\-off voor edge deployment?
- Benchmark: Vergelijking met een geïmplementeerd Python/PyTorch semi\-supervised model als referentie\.
- Semi\-supervised Effectiviteit: Hoe veel verbetert automatische pseudo\-labeling de accuracy t\.o\.v\. alleen gelabelde data?

## <a id="_nsnya9b1x3r6"></a>Wat zijn de minimale onderdelen waaruit de app bestaat?

- Data Pipeline
	- PlantVillage dataset loader \(download \+ lokale cache\)
	- Train/validation split \(80/10/10\), semi\-supervised labeling simulator \(mark 70\-80% as unlabeled\)
	- Data augmentation \(rotation, brightness, contrast, crop\) voor semi\-supervised robustness
- Rust \+ Burn Implementation
	- CNN architecture opgebouwd met Burn \(convolutional layers, pooling, fully connected\)
	- Forward pass voor inference

	- Model quantization/optimization \(TensorFlow Lite conversion of Burn\-native optimization\)
	- Jetson Orin Nano deployment script \(CUDA setup, model loading, inference\)
- Monitoring & Retraining

	- Trigger retraining quando pseudo\-label quality drops or every N images

## <a id="_1f3vi2p98fli"></a>Hoe zorg je ervoor dat jouw applicatie voldoende relevant is?

Relevantie: Deze applicatie lost een praktisch probleem op in landbouw/tuinbouw: gewasbewaking in real\-time zonder cloud\-afhankelijkheid\. Semi\-supervised learning is essentieel omdat handmatige labeling van duizenden bladfoto's onhaalbaar is\. Door Rust \+ Burn in te zetten, bereiken we:

1. __Snelheid__: Inference <200ms op Jetson \(real\-time bruikbaar\)
2. __Privacy__: Alle data blijft lokaal, geen cloud\-upload nodig
3. __Autonomie__: Model leert bij via pseudo\-labeling, ontlasten van handmatig labelen
4. __Betrouwbaarheid__: Offline werking, geen internetconnectie vereist
5. __Toekomstbestendig__: Kan uitgebreid worden naar andere ziekteclassificaties met CVAT \+ Roboflow workflow

Deze combinatie maakt het geschikt voor praktische deployment in gekste, real\-world landbouwomgevingen\.

## <a id="_ylyjtc5p5p0"></a>Welke technische uitdagingen verwacht je?

- Burn library ontkennen: Burn is nog relatief jong, minder StackOverflow support dan PyTorch\. Learning curve voor GPU backend setup\.
- Semi\-supervised label noise: Pseudo\-labels kunnen fout zijn → noise accumulation in training\. Mitigatie: confidence thresholds, periodic validation op gold\-standard labels\.
- Edge hardware constraints: Jetson Orin Nano heeft beperkt VRAM \(8GB\)\. Model moet klein genoeg zijn zonder accuracy verlies\. Trade\-off tuning nodig\.
- Retraining frequentie puzzle: Hoe vaak retrainen? Te vaak = computational waste, te weinig = model drift\. Experimenteel bepalen\.
- Deployment reproducibility: Burn versies, CUDA versies, Linux kernel differences op Jetson kunnen build issues veroorzaken\.

## <a id="_9lpnbdr36sg"></a>Wanneer is het onderzoeksproject geslaagd?

Het project is geslaagd wanneer de volgende resultaten zijn bereikt:

- Een volledig functioneel semi\-supervised CNN is geïmplementeerd in Rust met Burn, getraind op PlantVillage \(39 classes\), met 20\-30% gelabelde data \(semi\-supervised scenario\)\.
